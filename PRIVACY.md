# Политика конфиденциальности — WhatsApp Bulk Sender

**Последнее обновление:** 14 апреля 2026 г.

---

## 1. Общие положения

Настоящая Политика конфиденциальности описывает, какую информацию собирает,
использует и хранит расширение браузера Google Chrome **«WhatsApp Bulk Sender»**
(далее — «Расширение»).

Используя Расширение, Вы соглашаетесь с условиями настоящей Политики.

---

## 2. Какие данные собираются

Расширение **НЕ собирает и НЕ передаёт** персональные данные на внешние серверы.

Все данные хранятся **локально на устройстве пользователя** через Chrome Storage API
(`chrome.storage.local`) и **никогда** не покидают устройство пользователя,
за исключением случаев, описанных в разделе 5.

### 2.1. Данные, хранящиеся локально:

| Данные | Назначение | Срок хранения |
|---|---|---|
| Список номеров телефонов получателей | Для отправки сообщений через WhatsApp Web | До ручного удаления пользователем |
| Имена получателей (если указаны) | Для подстановки `{name}` в шаблоны | До ручного удаления |
| Шаблоны текстовых сообщений | Для повторного использования | До ручного удаления |
| Сохранённые очереди номеров | Для повторного использования | До ручного удаления |
| Прикреплённые медиа-файлы (фото/видео) | Для прикрепления к рассылке | До ручного удаления |
| Настройки (задержка, язык) | Для работы расширения | До ручного удаления |
| Прогресс и журнал рассылки | Для отображения статистики | До ручного удаления |

### 2.2. Что НЕ собирается:

- ❌ Личные данные пользователя (имя, email, телефон аккаунта)
- ❌ История переписки в WhatsApp
- ❌ Учётные данные, пароли, cookies, токены
- ❌ Геолокация
- ❌ Данные о других сайтах, посещаемых пользователем
- ❌ Аналитика, телеметрия, рекламные идентификаторы

---

## 3. Как используются данные

Введённые пользователем данные используются **только** для выполнения функций Расширения:
- отправка текстовых сообщений и медиа через открытую вкладку `web.whatsapp.com`
- подстановка имени получателя в шаблон
- отображение статистики рассылки
- сохранение шаблонов и очередей для повторного использования

Никакой обработки, агрегирования, анализа или продажи данных не производится.

---

## 4. Передача данных третьим лицам

Расширение **не передаёт** пользовательские данные третьим лицам, рекламным сетям,
аналитическим системам или разработчику.

---

## 5. Внешние сетевые соединения

Расширение выполняет ограниченный набор внешних соединений, **не содержащих**
пользовательских данных:

### 5.1. Загрузка библиотеки WhatsApp API (`wa-js`)

При первом открытии `web.whatsapp.com` Расширение загружает open-source библиотеку
`@wppconnect/wa-js` с публичного CDN:

```
https://cdn.jsdelivr.net/npm/@wppconnect/wa-js@3.23.3/dist/wppconnect-wa.js
```

Библиотека необходима для взаимодействия с внутренним API WhatsApp Web. При этой
загрузке передаются только стандартные HTTP-заголовки браузера
(User-Agent, IP). Номера, тексты и вложения **не отправляются**.

Исходный код библиотеки доступен по адресу:
<https://github.com/wppconnect-team/wa-js>

### 5.2. Проверка обновлений

Раз в 6 часов Расширение запрашивает файл `version.json` с настроенного сервера
обновлений (GitHub Raw или собственный сервер разработчика), чтобы узнать текущую
актуальную версию. Запрос содержит только HTTP-заголовки браузера; пользовательские
данные не передаются.

### 5.3. Отправка сообщений через WhatsApp Web

Сами сообщения отправляются **через официальный веб-интерфейс** WhatsApp Web
(`web.whatsapp.com`) в контексте Вашего аккаунта WhatsApp — так же, как если бы
Вы отправляли их вручную. Разработчик Расширения **не имеет доступа** к Вашему
аккаунту, номеру, переписке или содержимому сообщений.

## 6. Разрешения расширения

Расширение запрашивает следующие разрешения и использует их исключительно
для перечисленных целей:

| Разрешение | Назначение |
|---|---|
| `storage` | Локальное хранение шаблонов, очередей, прогресса |
| `tabs` | Поиск открытой вкладки `web.whatsapp.com` |
| `scripting` | Выполнение content-скрипта в контексте WhatsApp Web |
| `declarativeNetRequest` | Удаление CSP-заголовков **только** на `web.whatsapp.com` для работы библиотеки `wa-js` |
| `alarms` | Периодическая проверка обновлений раз в 6 часов |
| `host_permissions: https://web.whatsapp.com/*` | Единственный домен, на котором работает расширение |

Расширение **не имеет доступа** к другим сайтам, которые Вы посещаете.

---

## 7. Безопасность

- Все пользовательские данные хранятся в зашифрованном профиле Chrome.
- Расширение **не использует** внешние серверы для хранения данных.
- Открытый исходный код расширения доступен для аудита:
  <https://github.com/Mob2222/whatsapp-sender>

---

## 8. Права пользователя

Вы можете в любой момент:
- **Удалить все сохранённые данные**, удалив расширение через `chrome://extensions`
  (Chrome при удалении автоматически стирает `chrome.storage.local`).
- **Очистить отдельные шаблоны или очереди** внутри интерфейса расширения
  (кнопка ✕ рядом с элементом).
- **Отключить авто-проверку обновлений**, удалив расширение или отключив его в
  `chrome://extensions` (переключатель «Включено»).

## 9. Дети

данные от детей. Если Вы младше 18, пожалуйста, не используйте Расширение.

## 11. Изменения в Политике

При внесении изменений в настоящую Политику обновлённая версия публикуется по

в релизных заметках Расширения.
